<h1 align="center">App Modernization Starter Pack</h1>

A starter pack demonstrating how GitHub Copilot can accelerate legacy application modernization — from architecture analysis through phased migration to modern CI/CD and packaging.

## Featured Reference

| | |
|---|---|
| **Repository** | [samqbush/doomv2](https://github.com/samqbush/doomv2) |
| **What it demonstrates** | Using GitHub Copilot to modernize a 1997 C codebase (DOOM Linux 1.10) — dead toolchain resurrection, phased platform migration, AI-generated architecture docs, automated testing, CI/CD, and release packaging |
| **Key Copilot capabilities shown** | Architecture analysis, modernization planning, code generation for platform rewrites, copilot-instructions.md for project context, automated CI/CD workflows |

## How to Use

1. **Evaluating Copilot for modernization?** Start with the Overview, then review the Reference Walkthrough.
2. **Ready to modernize?** Use the Playbook to structure your phased approach.
3. **Setting up Copilot context?** Adapt the Copilot Instructions Best Practices for your codebase.
4. **Reporting to leadership?** Use Measuring Success to quantify impact.
