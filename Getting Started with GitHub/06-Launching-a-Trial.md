# Launching a Trial

## Overview

GitHub offers a **free 30-day trial** of GitHub Enterprise Cloud with no credit card required. This guide walks you through getting your OpCo started with a trial so you can evaluate the platform with real users and workflows.

---

## Trial Details

| Detail | Info |
|--------|------|
| **Duration** | 30 days |
| **Users** | Up to 50 seats |
| **Payment required** | No credit card needed |
| **Features included** | Most Enterprise Cloud features |
| **Not included** | GitHub Codespaces, Copilot Enterprise (separate trials available) |
| **After trial** | Upgrade to paid plan or downgrade to Team/Free |

---

## Step-by-Step: Start a GitHub Enterprise Cloud Trial

### Prerequisites
- A personal GitHub account (create one at [github.com/join](https://github.com/join))
- Company email address for verification

### Steps

**1. Go to the Enterprise Trial Page**
- Visit [github.com/enterprise](https://github.com/enterprise)
- Click **"Start a free trial"**

**2. Complete the Setup Form**
- Enter your desired **enterprise name** (e.g., `yourcompany-trial`)
- Provide company information (name, size, industry)
- Accept GitHub's Evaluation License Agreement

**3. Verify Your Account**
- Select your country
- Complete the verification task
- Accept terms and conditions

**4. Create Your Enterprise**
- Click **"Create enterprise"**
- Your 30-day trial begins immediately

**5. Set Up Your First Organization**
- You can create up to 3 organizations during the trial
- Or invite existing organizations into your trial enterprise

**6. Add Users**
- Invite team members by email or GitHub username
- Up to 50 users can participate in the trial

## What to Evaluate During Your Trial

### Week 1: Setup & Basics
- [ ] Configure SSO with your Identity Provider
- [ ] Create teams and set permissions
- [ ] Import or create test repositories
- [ ] Set up branch protection rules
- [ ] Invite initial users (5–10 developers)

### Week 2: Developer Workflows
- [ ] Set up GitHub Actions CI/CD pipelines
- [ ] Test pull request workflows and code review
- [ ] Explore GitHub Projects for planning
- [ ] Try GitHub Codespaces for cloud development

### Week 3: Security & Compliance
- [ ] Enable secret scanning on repositories
- [ ] Set up code scanning with CodeQL
- [ ] Review the Security Overview dashboard
- [ ] Configure audit log streaming (optional)
- [ ] Test Dependabot alerts and updates

### Week 4: Advanced Features & Decision
- [ ] Evaluate Copilot (separate trial available)
- [ ] Test admin controls and policy management
- [ ] Gather team feedback
- [ ] Assess readiness for paid deployment
- [ ] Engage your GitHub account team for pricing

## Copilot Trial (Separate)

GitHub Copilot Business offers its own trial:
1. Go to your organization settings → **Copilot**
2. Start the Copilot Business trial
3. Assign seats to developers
4. Trial lasts 30 days

---

## Enterprise Server Trial

If your OpCo needs an on-premises evaluation:
1. Visit [enterprise.github.com/trial](https://enterprise.github.com/trial)
2. Request a trial license
3. Download and deploy GHES on your infrastructure
4. Trial license is valid for 45 days

---

## After the Trial

| Option | What Happens |
|--------|-------------|
| **Upgrade to Enterprise** | Contact your GitHub account rep or purchase online |
| **Downgrade to Team** | Retains repositories; loses enterprise features |
| **Downgrade to Free** | Most features removed; repos remain accessible |
| **Let trial expire** | Enterprise features disabled; data retained for 90 days |

---

## Tips for a Successful Trial

1. **Start small** – Begin with 5–10 engaged developers, expand as you learn
2. **Use real workflows** – Import actual projects to test realistic scenarios
3. **Engage stakeholders** – Include security, ops, and management in evaluation
4. **Document findings** – Track what works and what needs configuration
5. **Talk to GitHub** – Your account team can provide trial extensions and technical support
