<h1 align="center">Platform Engineering Resource Pack</h1>

A practical resource pack for teams building reusable developer platforms on GitHub. Use it to define your golden paths, standardize repository scaffolding, automate controls, and create self-service experiences that help engineers ship faster with less cognitive load.

## Contents

| # | Document | Description |
|---|----------|-------------|
| 01 | [Platform Engineering on GitHub](./01-Platform-Engineering-on-GitHub.md) | Foundations, operating model, and how GitHub supports an internal developer platform |
| 02 | [Template Repositories](./02-Template-Repositories.md) | Patterns for reusable repository scaffolds, archetypes, and automation |
| 03 | [Repository Rulesets and Policies](./03-Repository-Rulesets-and-Policies.md) | How to scale governance, compliance, and consistency with GitHub controls |
| 04 | [GitHub Apps and Automation](./04-GitHub-Apps-and-Automation.md) | Custom apps, Actions, webhooks, and workflow automation for platform teams |
| 05 | [Internal Developer Portal](./05-Internal-Developer-Portal.md) | Portal, catalog, and self-service patterns for discoverability and adoption |

## How to Use

1. **Starting your platform journey?** Read [01](./01-Platform-Engineering-on-GitHub.md) first to align on outcomes, golden paths, and team responsibilities.
2. **Standardizing software delivery?** Use [02](./02-Template-Repositories.md) and [03](./03-Repository-Rulesets-and-Policies.md) together to define paved roads and guardrails.
3. **Automating the platform?** Apply [04](./04-GitHub-Apps-and-Automation.md) to build self-service workflows, policy enforcement, and operational tooling.
4. **Improving discoverability?** Use [05](./05-Internal-Developer-Portal.md) to design a developer portal and service catalog experience.
5. **Adapting to your organization?** Copy the tables, examples, and checklists into your own engineering standards, templates, and enablement docs.
